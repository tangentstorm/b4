** TODO dependencies between tests
It would be kind of cool to document the dependencies between tests,
and use them to generate pert charts, in the following way:

- the b4vm implementation really doesn't need to be 100% complete
until you start needing to actually use the opcodes.

You should be able to implement a spike solution that works at
multiple layers.

You should be able to implement the opcodes in any order you
like, for the most part... But some of them might require other
opcodes (like =ad= needs operands to add, so you need =lb= or =c1=
or similar)

But even as b4 is easy to implement, you shouldn't have to implement
it ALL before moving on to the next thing.

** IDEA posix-friendly "block storage" files
Forth systems tend to use "blocks" instead of "files"
- simplifies editors (no scrolling, just edit screen)

I made a lil/decker tool to browse old block files.
- [ ] find the url?

Idea to make block format that's readable with modern tools:
- force 64-char lines with lf
- extra lf or ---- line every 64 lines
- loader would just ignore those lines
- saver would always add those lines
- use a blkfmt file to enforce/repair the format
